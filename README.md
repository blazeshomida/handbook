# Handbook

This repo contains reusable engineering standards and copyable file templates
for code, documentation, workflow, and release work.

## Contents

- [Standards](standards/README.md): durable guidance for writing and reviewing
  code, documentation, commits, pull requests, versions, and releases.
- [Templates](templates/README.md): copyable file shapes for project agent
  instructions, pull requests, changesets, and release notes.
- [Agent instructions](AGENTS.md): editing rules for agents and contributors
  working in this repo.

## Structure

```txt
standards/
  code/
  workflow/
templates/
  agents/
  github/
  releases/
```

## How To Use This Handbook

Use standards when deciding how to write or review something. Start with the
standard that owns the decision, then follow links to related standards.

Use templates when a project needs a copyable file. Replace placeholders after
copying. Template links point back to the canonical handbook standards on
GitHub.

## Boundaries

This repo does not contain full project templates, generated starter apps, or
tool configuration presets. Those belong in a separate repo or package.
